# Reflection boi loop

Status: scaffolded

## Purpose
Turn execution outcomes, research notes, and session logs into reusable learning artifacts.

## Inputs
- Latest session note
- Trade fills and broker state
- Research notes and watchlist changes
- Any signal misses, rejects, or surprises

## Outputs
- A short reflection on what happened
- A signal-quality note on what mattered
- A list of follow-up actions
- A capture of reusable patterns for the learning folder

## Loop
1. Read the newest session log.
2. Capture what worked, what failed, and what changed.
3. Separate one-off noise from repeatable signals.
4. Write the distilled takeaways into the learning folder.
5. Feed any durable conclusions back into research, execution, or risk docs.
6. Close the loop by updating the session log with the new learning reference.

## Guardrails
- Keep the reflection short and factual.
- Avoid overwriting source session notes; append learning artifacts instead.
- Promote only durable patterns back into core repo docs.
